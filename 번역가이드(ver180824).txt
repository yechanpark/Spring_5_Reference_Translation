* 배포 금지, 개인용도로만 사용, 무단배포 시 불이익 개인책임

1. 번역 방식
영어 한 문단 당 한글 한 문단
소스 코드 예시는 건들지 않음

2. 템플릿
스프링 레퍼런스 css랑 똑같은 템플릿 사용
영문단을 번역하여 문단 다음에 html 태그를 추가하고 번역한 내용을 기술하는 방식

3. 사용 툴
winmerge - 이전 버전이랑 최신버전이랑 달라진거 비교하기용
github - 각각 해석한거 머지하기용
edit tool - 스크립트 작성은 자유

4. 그 외
html 스크립팅 시 들여쓰기
번역하기 애매한건 상의하거나 우선 영어로 적어놓음 (Context 등)
resource(.js, .png, .css 등) 파일명과 경로는 레퍼런스 문서에서 쓰는 그대로

5. 브랜치
각 레퍼런스 문서마다 브랜치를 따서 진행 (필수아님, 권장)
ex1 ) SpringSecurity 관련 작업을 할 때와 SpringData 관련 작업을 할 때는 브랜치를 따로 관리하는 것을 권장
ex2 ) SpringData 중에서도 Hibernate, JPA 등 다른 분류에 대해 작성할 때도 따로 관리하는 것을 권장
